# Repo structure

This file is copied into every project bootstrapped via `/start-project`. It codifies the universal `docs/` layout. Stack-specific layout (e.g., `src/`, `tests/`, `notebooks/`) belongs to the project's L3 template, not this file.

Source: `~/Projects/cascade-system/docs/structure.md` (this is the downstream copy with `retros/` placement adjusted).

## Tree

```
<project>/
├── README.md
├── .windsurf/
│   ├── rules/                      # L2 overrides (project-specific rules)
│   ├── skills/                     # L2 overrides (project-specific skills)
│   ├── workflows/                  # L2 overrides (project-specific workflows)
│   └── phases.yaml                 # phase taxonomy for this project type
└── docs/
    ├── structure.md                # this file
    ├── architecture/
    │   ├── *.excalidraw            # conceptual diagrams
    │   └── *.mmd                   # Mermaid diagrams
    ├── decisions/
    │   ├── INDEX.md                # ADR index
    │   └── ADR-NNN-<slug>.md       # one per significant decision
    ├── handoffs/
    │   ├── INDEX.md                # cross-Cascade handoff prompts
    │   └── <from-cascade-id>-<topic>.md
    ├── retros/                     # project-local retros (per-milestone or per-sprint)
    │   └── <milestone-slug>.md
    └── prompts/
        └── stages/                 # one file per phase from phases.yaml
            └── NN-<phase-slug>.md
```

## Placement rules

1. **Every artifact lives in its canonical subdirectory.** `docs/` root holds only `structure.md` (and project-specific cross-cutting docs like `technical.md` or `makefile-manual.md` if relevant).
2. **New top-level paths require an ADR.** Precedent: cascade-system's `ADR-002-handoff-prompts-subdirectory.md`.
3. **ADRs are immutable** once `Status: Accepted`. Override via `Status: Superseded by ADR-MMM`. Never delete.
4. **Indexes** (`INDEX.md` in `decisions/` and `handoffs/`) are regenerated by `/docs-refresh` (when L1 ships it). Update manually until then.
5. **Retros are local** to this project at `docs/retros/`. (The cascade-system meta-repo keeps retros at root `retros/` because they aggregate cross-project learnings — that's specific to the meta-repo, not this project.)

## File-naming conventions

| Subdirectory | Pattern |
|---|---|
| `docs/decisions/` | `ADR-NNN-<kebab-slug>.md` (NNN zero-padded) |
| `docs/handoffs/` | `<cascade-id>-<topic>.md` |
| `docs/retros/` | `<milestone-slug>.md` or `<phase-name>.md` |
| `docs/architecture/` | `*.excalidraw` or `*.mmd` |
| `docs/prompts/stages/` | `NN-<phase-slug>.md` (NN matches phase order in `phases.yaml`) |

## Validators

`/docs-refresh` (when ready) regenerates `INDEX.md` files and rejects placement violations. Until then, placement discipline is manual; see `.windsurf/rules/strict-docs-placement.md` for project-local enforcement and `~/.windsurf/rules/clean-project-structure.md` for the universal L1 rule.
